## Dev Commands

```bash
npm run dev              # Start dev server (localhost:3000)
npm run build            # Production build (CSS_TRANSFORMER_WASM=true set in package.json)
npm run typecheck        # tsc --noEmit
npm run lint             # ESLint (next/core-web-vitals + typescript)
npm run test             # Vitest watch mode
npm run test:run         # Vitest once
npx vitest run path/to/file.test.ts        # Single test file
npx vitest run -t "test name"              # Single test by name
npm run test:e2e         # Playwright (requires server already running)
npm run db:generate      # Generate Drizzle migration
npm run db:migrate       # Apply migrations
npm run db:push          # Push schema directly (dev only)
npm run db:seed          # Seed admin + customer + repair requests + products
npm run db:studio        # Drizzle Studio GUI
```

**CI order:** lint ‚Üí typecheck ‚Üí test ‚Üí build (enforced in `.github/workflows/ci.yml`)

## Setup

1. Copy `.env.example` to `.env.local` and fill in values
2. `npm install` ‚Äî triggers `.scripts/postinstall.js` which patches `lightningcss` WASM require path (needed for Next.js 16 builds)
3. `npm run db:push` ‚Äî push schema to Neon DB
4. `npm run db:seed` ‚Äî create test users (`admin@kitfix.co.za` / `KitFix2024!`, `customer@test.co.za` / `Test1234!`)

`.npmrc` has `legacy-peer-deps=true`.

## Conventions

- **Path alias:** `@/` maps to project root (`import { db } from "@/lib/db"`)
- **Monetary values:** Always ZAR cents (integers). Use `formatCurrency()` from `lib/utils.ts` to display.
- **Dates/times:** Use `formatDateSAST()` from `lib/utils.ts` ‚Äî `Africa/Johannesburg` (UTC+2, no DST).
- **CSS class merging:** Use `cn()` from `lib/utils.ts` (clsx + tailwind-merge).
- **Logging:** Use `logger` from `lib/logger.ts` ‚Äî never raw `console.*`. JSON in production, coloured in dev.
- **Unused variables:** Prefix with `_` to suppress ESLint.
- **Component library:** shadcn/ui `new-york` style. Add with `npx shadcn@latest add <component>`.

## Server Actions

All server actions return `ActionResult<T>`:
```ts
type ActionResult<T> =
  | { success: true; data: T }
  | { success: false; error: string; fieldErrors?: Record<string, string[]> };
```

Import this and all domain types from `@/types`, not directly from `@/lib/db/schema`.

## Auth Pattern

Auth uses **JWT (jose)** with a custom proxy ‚Äî not Better Auth directly for session checks.

- **Edge:** `proxy.ts` validates JWT from `better-auth.session_token` cookie, sets `x-user-id` / `x-user-role` / `x-session-id` headers.
- **Server components/actions:** Use `getSessionFromHeaders()` from `lib/auth-utils.ts` (reads proxy headers, no DB query). For full session with DB user row, use `getSession()`.
- **Auth helpers:** `requireAuth()`, `requireRole(roles)`, `requireOwnership(ownerId)` ‚Äî all in `lib/auth-utils.ts`. These redirect on failure.
- **Server action wrappers:** `authenticatedAction()`, `authenticatedAdminAction()`, `authenticatedRoleAction()` ‚Äî return `{ success: false, error: "..." }` instead of throwing.
- **Client-side:** `useSession()` from `lib/auth-client.ts` (custom hook, not Better Auth's).

## Database

- **ORM:** Drizzle with Neon serverless (`@neondatabase/serverless` + `drizzle-orm/neon-http`).
- **Schema:** `lib/db/schema.ts` ‚Äî all tables, enums, relations, inferred types.
- **DB client:** `lib/db/index.ts` ‚Äî lazy-init Proxy wrapper (never fails at import time).
- **Queries:** `lib/db/queries/` ‚Äî reusable query functions per domain (repairs, payments, reviews, notifications, orders, products, voice-notes).
- **Migrations:** `drizzle/` directory, generated by `drizzle-kit generate`.
- **Soft deletes:** `repairRequests.deletedAt` ‚Äî always filter with `isNull(deletedAt)` in queries.
- **11-stage pipeline:** `submitted ‚Üí reviewed ‚Üí quote_sent ‚Üí quote_accepted ‚Üí item_received ‚Üí in_repair ‚Üí quality_check ‚Üí ready_for_shipment ‚Üí shipped ‚Üí delivered ‚Üí cancelled`

## Testing

- **Vitest** with globals enabled (no need to import `describe`/`it`/`expect`).
- **Setup:** `vitest.setup.ts` mocks `next/cache` (`revalidatePath`, `revalidateTag`) and `next/headers` (`cookies`, `headers`).
- **Location:** Tests in `__tests__/` directories adjacent to source (e.g., `lib/validators/__tests__/`, `lib/courier/__tests__/`, `lib/shipping/__tests__/`).
- **Playwright:** Config in `playwright.config.ts`. Tests in `e2e/` directory. Requires dev server running. Uses Desktop Chrome only.

## Key Modules

- `lib/courier/` ‚Äî Courier Guy Locker API client (deep module: `client.ts`, `types.ts`, `index.ts`). Env: `TCG_API_KEY`.
- `lib/shipping/` ‚Äî Locker lookup and rate calculation (`lockers.ts`, `rates.ts`).
- `lib/config/pricing.ts` ‚Äî Pricing in ZAR cents (metro vs non-metro pickup/delivery fees).
- `lib/validators/` ‚Äî Zod schemas for form validation (repair, profile, review). Tests in `__tests__/`.
- `lib/env.ts` ‚Äî Lazy env validation via Zod Proxy. Skipped when `SKIP_ENV_VALIDATION=1` or `NODE_ENV=test`.
- `lib/email.ts` ‚Äî Resend email client. Fire-and-forget (never throws). Silently skips if `RESEND_API_KEY` missing.

## PWA

Manual service worker at `public/sw.js`, registered by `ServiceWorkerRegistrar` client component. **Not** using `next-pwa` (incompatible with Next.js 16 / Turbopack).
